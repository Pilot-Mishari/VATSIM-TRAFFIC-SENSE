1. System architecture
VATSIM API
     ↓
Collector Service
     ↓
PostgreSQL Database
     ↓
Backend API
     ↓
Frontend Dashboard

2. Architecture Components
    A. Storing Service
        fetch VATSIM data,
        parse traffic,
        taffic information,
        track controller sessions.
        interval 15 minutes
    B. SQL Database
        traffic history,
    airport statistics,
    controller sessions,
    event data,
    prediction data.
    C. backend
        Process data to frontend
            Node.js
            Express
            TypeScript
    D. Frontend
        airport analytics,
        traffic predictions,
        high traffic prediction timeline,
        controller coverage,
        controller activity timeline,
        historical trends,
        top-down airports recommendations

3. Database Design
    Airports:   
        id
        icao
        name
        country
        fir
        popularity_score

    Traffic Data:
        id
        airport_icao
        timestamp
        arrivals
        departures
        overflights
        total_aircraft
        traffic_score

    Controller Sessions:
        id
        callsign
        position_type
        airport_icao
        start_time
        end_time
        duration_minutes

    Active controllers:
        id
        callsign
        position
        airport_icao
        online_since

    Airport predictions:
        id
        airport_icao
        prediction_time
        predicted_traffic
        confidence_level
        predicted_staffing
